## Description
####################################################################################################

We officially start off on 11/12/2022 with our first coding problem. We will solve an algo problem
everyday. I mean every single day. That is the only way we are going to get good.
It will start off hard but it will become easier. You just have to make it a routine/habit and your
will definitely gain from it.

####################################################################################################
## Preparation
####################################################################################################
You need a github account and github knowledge because this is how resources will be shared.

####################################################################################################
## Approach
####################################################################################################

We are going to solve the algo problems by category. The categories have questions with different
difficulty levels namely:

categories:
    arrays (24)
    linked Lists (13)
    strings (15)
    sorting (9)
    stacks (7)
    binary_search_trees (10)
    binary_trees (13)
    searching (7)
    graphs (13)
    recursion (13)
    heaps (5)
    tries (2)
    greedy_algorithms (5)
    famous_algorithms (5)
    dynamic_programming (19)

difficulty:
    easy,
    medium,
    hard and
    very hard

We will sort the problems in each category following the order above respectively.

I will share a problem everyday. Give your self maximum 30 minutes to attempt solving the problem without
looking for the solution.
This technique will pose a challenge to your brain hence help build the brain muscle.  
If the problem it too hard for you, don't worry.
Remember we have the solutions to all the questions so don't waste more than 30 minutes.  
At the end of the day, we will post the solution to the public github repository and we can also arrange
time for discussion. We can have 1 hour or 2 every Saturday for discussion. This is food for thought.

We will have 2 solutions. The bruteforce solution and the optimized solution. During coding interviews,
your supervisor will always ask you: How can you make your code better?
That is where the optimized solution comes in.

I advice you to read about space and time complexity. These aren't complecated topics.
We will also explain them in our first algo meeting.
